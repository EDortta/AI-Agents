# 11 - Fundamentos de Gestão de Issues

## História feliz: menos ruído, mais entrega
Lia percebeu que não adiantava só codar rápido. Sem boa gestão de issue, o time perdia contexto, duplicava trabalho e quebrava escopo.

## O que é seu (programador)
- Transformar pedido em issue clara (objetivo, escopo, risco, teste, DoD).
- Garantir vínculo com `work_id`.
- Manter status real (não otimista).

## O que é do agente
- Ajudar a estruturar issue com padrão.
- Sugerir descrição completa para Jira/GitHub.
- Organizar docs locais de épico/tarefa com consistência.

## Estrutura mínima de issue boa
- Contexto
- Objetivo
- In scope
- Out of scope
- ARO (Acceptance, Risk, Operations)
- Test plan
- DoD

## Regra prática
Issue ruim gera execução ruim. Invista 5-10 minutos na issue e economize horas de retrabalho.
