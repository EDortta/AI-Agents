# 13 - Offline Issue Flow (Local Mode)

## Happy story: no internet, no freeze
Lia lost external access but still progressed using local issue workflow with full traceability.

## What is yours (programmer)
- Organize epics/tasks in `docs/issues/`.
- Update status via rename and evidence.
- Prepare later sync package for Jira/GitHub.

## What is the agent's
- Keep local issue structure consistent.
- Produce clean handoff for restart.
- Suggest sync checklist for when online access returns.

## Offline step by step
1. Create/update local issue docs.
2. Fill `work_id`, date, and test plan.
3. Implement in small stages.
4. Run session-close (`handoff.md` + `napkin-lessons`).
5. Record pending external sync items.

## When back online
- Create Jira/GitHub issues from local docs.
- Reuse the same `work_id` for traceability.
- Update final links in docs and handoff.
