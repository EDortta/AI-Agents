# 03 - How to Use handoff.md

## Happy story: clean session, fast restart
At session end, Lia writes handoff. Next day she resumes in minutes without depending on chat history.

## What is yours (programmer)
- Update `handoff.md` at end of each stage.
- Keep next steps actionable.
- Record blockers honestly.

## What is the agent's
- Draft handoff from completed work.
- Confirm changed files and executed checks.

## Minimum handoff entry
- date + work_id
- current status
- next steps
- blockers/risks
- changed files
- checks/tests executed
