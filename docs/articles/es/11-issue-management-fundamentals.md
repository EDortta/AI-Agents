# 11 - Fundamentos de Gestión de Issues

## Historia feliz: menos ruido, mejor entrega
Lia entendió que no alcanza con programar rápido. Sin buenas issues, el equipo pierde contexto, duplica trabajo y rompe alcance.

## Qué es tuyo (programador)
- Convertir solicitudes en issues claras (objetivo, alcance, riesgo, tests, DoD).
- Mantener vínculo con `work_id`.
- Mantener estado realista.

## Qué es del agente
- Ayudar a estructurar issues con plantillas.
- Redactar cuerpos completos para Jira/GitHub.
- Mantener consistencia en docs locales de épica/tarea.

## Estructura mínima de una buena issue
- Contexto
- Objetivo
- In scope
- Out of scope
- ARO (Acceptance, Risk, Operations)
- Test plan
- DoD

## Regla práctica
Issue débil genera ejecución débil. Invierte minutos aquí y ahorra horas después.
