# 13 - Flujo Offline de Issues (Modo Local)

## Historia feliz: sin internet, sin bloqueo
Lia perdió acceso externo y aun así avanzó usando flujo local de issues con trazabilidad.

## Qué es tuyo (programador)
- Organizar épicas/tareas en `docs/issues/`.
- Actualizar estado por renombre y evidencia.
- Preparar sincronización posterior con Jira/GitHub.

## Qué es del agente
- Mantener estructura local consistente.
- Generar handoff claro para retomar.
- Sugerir checklist de sincronización.

## Paso a paso offline
1. Crear/actualizar docs locales de issue.
2. Completar `work_id`, fecha y plan de tests.
3. Implementar en etapas cortas.
4. Ejecutar session-close (`handoff.md` + `napkin-lessons`).
5. Registrar pendientes de sincronización externa.

## Al volver online
- Crear issues Jira/GitHub desde docs locales.
- Reutilizar el mismo `work_id`.
- Actualizar links finales en docs y handoff.
